# Kirby Legaltext

Kirby Legaltext adds a `legaltext()` field method for repairing table-of-contents anchor links in generated legal text.

## Problem

Privacy policies from [datenschutz-generator.de](https://datenschutz-generator.de/) include a table of contents with local anchor links, for example `href="#privacy"`. Those links point to `id` attributes on headings later in the document.

When that generated content is pasted into a Kirby Writer field, the Writer field removes the `id` attributes from the headings. The table-of-contents links remain in the text, but their targets no longer exist in the rendered page. As a result, clicking the ToC links does not jump to the intended sections.

## Solution

This plugin renders the field with KirbyText, looks for the generated `Table of contents` section, reads its local anchor targets, and restores those IDs onto matching headings in the rendered HTML.

Use the `legaltext()` field method instead of rendering the field directly when outputting this kind of generated legal text.

## Requirements

- Kirby 4 or 5
- PHP 8.1 or newer
- PHP DOM and libxml extensions

## Installation

Add the GitHub repository to your project's `composer.json`:

```json
{
  "repositories": [
    {
      "type": "vcs",
      "url": "git@github.com:presprog/kirby-legaltext.git"
    }
  ]
}
```

Then install the plugin package with Composer:

```sh
composer require presprog/kirby-legaltext
```

## Usage

Use the `legaltext()` field method in a template:

```php
<?= $page->text()->legaltext() ?>
```

The method should be used on the field that contains the generated legal text.

## Expected Content Shape

The expected content shape derives from the HTML generated by [datenschutz-generator.de](https://datenschutz-generator.de/). The plugin expects rendered HTML with:

- a heading named `Table of contents`, currently detected as `h2` or `h3`
- local ToC links like `href="#some-section"`
- matching section headings later in the document

It first tries to match ToC entries to headings by normalized heading text. If that fails, it falls back to matching entries to headings at the same heading level by position.

## Testing

Install the plugin development dependencies:

```sh
composer install
```

Run the test suite from the plugin directory:

```sh
vendor/bin/phpunit
```

## Important Notice

This plugin was almost entirely AI generated, is experimental, and solves a very niche problem. It is probably of no use for you. It was made public to simplify using it in our own projects and to share it with the Kirby community.
